# Workshop Starting-Point Prep: Remove 3 Features (in reverse order)

## Goal
The current `main` (commit `de818b2`) is the **finished** workshop result. Produce a
**starting point** that lacks three features so participants can implement them in this
order: **(1) History Stacking → (2) View Transitions → (3) React Compiler + TanStack Virtual**.

We therefore **remove** in reverse order, one commit per feature:
1. React Compiler + TanStack Virtual
2. View Transitions
3. History Stacking

All app code lives in `webapp/`. Repo is on `main`, working tree clean. No test framework
exists — "test the code" = `pnpm typecheck`; manual testing is done by the **user** via `make dev`.

## Per-step workflow (interactive)
For each of the 3 steps:
1. Make the removal edits.
2. Run the automated check in `webapp/`:
   - Step 1 only: `pnpm install` (sync `pnpm-lock.yaml` after dep removal), then `pnpm typecheck`.
   - Steps 2 & 3: `pnpm typecheck`.
3. **PAUSE** and hand off to the user to manually test the app (`make dev`). Wait for their OK.
4. Commit the changes.
5. Create a revert patch in the **parent folder** (re-adds the feature):
   `git -C <repo> diff HEAD HEAD~1 > "/Volumes/Work/Check24/Repositories/<patch-name>.patch"`
   (Parent folder is outside the repo → automatically untracked by git.)
6. Proceed to the next step.

Patch direction = **re-adds the feature** (apply with `git apply <patch>` to restore the solution).
Commit message style: `chore(workshop): remove <feature>`.

Leave the design doc `webapp_demo_projekt_setup_3bf450b4.plan.md` **untouched** in all steps.

    ```tsx
      for (const [key, value] of Object.entries(newParams)) {
        if (value !== undefined && value !== "") next.set(key, value);
        else next.delete(key);
      }
      setSearchParams(next, { replace: true });
    };
    ```
